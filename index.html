60-second quickstart

No dependencies. No setup. No registration.

1. Install (Linux / macOS)

curl -fsSL https://github.com/jdp5949/rusty-sched/releases/latest/download/install.sh | sh

1. Install (Windows PowerShell)

irm https://github.com/jdp5949/rusty-sched/releases/latest/download/install.ps1 | iex

2. Start the server (single node)

rusty-sched server

Open http://localhost:8080 — that's it.

3. (Optional) Start an agent on every worker host

rusty-sched agent --servers your-server:7000

4. Define a job (YAML or via UI)

cat <<EOF | rusty-sched job apply -f -
name: nightly-etl
cron: "0 2 * * *"
cmd: /opt/etl/run.sh
target: { tag: etl }
retry: { max_attempts: 3, backoff: { kind: exponential, base_secs: 30, max_secs: 600 } }
timeout_secs: 3600
sla_secs: 1800
alerts:
  events: [on_failure, on_sla_miss]
  channels:
    - { kind: slack, webhook_url: "$SLACK_WEBHOOK" }
EOF

🛡️ Production-grade HA in 30 more seconds

Spin up a 3-node cluster — survives any single node failure with sub-second failover:

rusty-sched server --peers node1:7000,node2:7000,node3:7000

Everything Autosys can do, simpler

📅

Triggers

Cron, interval, one-shot, file watch, webhook, manual, dependency on upstream jobs.

🧩

Dependencies + Boxes

DAGs with AND/OR/regardless conditions. Autosys-style "box jobs" grouped as a unit. Nested.

🔁

Retries + SLA

Fixed / exponential backoff. Hard timeouts. SLA breach alerts. Late-start detection.

📆

Calendars

Business days, blackout dates, time-of-day windows. AND-combined rules.

🌐

Web UI

Live log tail. Gantt timeline. Dependency graph. Manual trigger. Pause/resume. Audit log.

🛰️

Multi-host agents

Lightweight agent on every host. mTLS gRPC. Tag-based routing. Real-time log streaming.

🏰

3-node HA cluster

Embedded Raft (openraft). Sub-second failover. Zero missed fires. No external DB.

📣

Alerts

Email (SMTP), Slack, generic webhook. Per-job + per-box + global routing.

🔐

Auth + RBAC

Local users, bcrypt passwords, session cookies, roles (admin/operator/viewer). Audit log.
