rusty-sched

Auth + RBAC + API keys

rusty-sched ships with bcrypt-hashed passwords, cookie sessions, API keys, three RBAC roles, and an audit log on every mutation.

First-run bootstrap

On startup, if zero users exist, the binary auto-creates admin with a password sourced from:

  1. RSCHED_ADMIN_PASSWORD env var (preferred).
  2. A randomly generated string that is logged at WARN level on stdout.
RSCHED_ADMIN_PASSWORD=change-me rusty-sched server

Roles

RoleReadWrite jobs / runs / globals / resourcesAdmin (users + audit + manage any key)
viewer✅❌❌
operator✅✅❌
admin✅✅✅

Enforced by axum extractors:

  • AuthUser — must be authenticated
  • RequireWrite — must be operator or admin
  • RequireAdmin — must be admin

Anonymous access is allowed for read endpoints. Write endpoints return 401 when no credentials are present, 403 when credentials lack the role.

Sessions (UI)

POST /api/v1/auth/login sets a rsched_session cookie:

rsched_session=<32-byte-base64url>; Path=/; HttpOnly; SameSite=Lax; Max-Age=43200

TTL: 12 hours. Pruned periodically via SessionRepo::prune_expired.

API keys (CLI / integrations)

Generated by the user in the API keys UI tab. The plaintext token is shown once on creation — you can't recover it later. Use it as a Bearer token:

export RSCHED_TOKEN=<paste-token>
rusty-sched cli list
curl -H "Authorization: Bearer $RSCHED_TOKEN" http://localhost:8080/api/v1/jobs

Tokens are bcrypt-hashed at rest. On every request the auth middleware loops over active keys for the current user and verifys against each hash. To keep this fast in deployments with many keys, the all_active query is filtered by expiry + disabled flags.

Audit log

Every successful mutation records an AuditEntry:

ActionTrigger
auth.loginSuccessful login
apikey.create / apikey.deleteAPI key lifecycle
user.createAdmin created a user
job.create / job.update / job.deleteJob mutations
job.pause / job.resume / job.triggerJob state changes
run.kill / run.signal / run.change_statusRun interventions
global.set / global.deleteGlobals
resource.create / resource.deleteVirtual resources

Admins can view the last 200 entries via GET /api/v1/audit or the Audit UI tab.
